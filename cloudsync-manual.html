前半：操作手順

⓪ システム概要（5〜10分）

CloudSyncは「機材が今どこにあるか・どの案件に出ているか」をリアルタイムで把握するためのシステムです。

  • これまで: Excelや口頭で管理 → 二重貸し出し・所在不明が発生しやすい
  • これから: アプリで出荷・返却を記録 → 誰でも今の在庫がわかる

今日のゴール：出荷・入庫の操作で止まらないこと。それだけです。

① RFIDタグ貼り付け＋機材紐付け

  1. まだRFIDが貼れていない機材を用意する
  2. タグをシールで貼る（貼り場所は固定位置を示す）
  3. アプリを開く → 機材登録 → タグをスキャン → 紐付け完了
「タグを貼ったら、その場でアプリでの紐付けまでセットでやってください。貼りっぱなしはNG。」
タンク・マニホールド・Y字管は貼り場所が未決定 → 今回は説明のみ、後日対応

② 出荷 + 入庫・返却 練習

出荷の手順

  1. アプリを開く
  2. 案件画面 → 対象タスクを選ぶ
  3. 機材のRFIDタグをスキャン
  4. ステータスを「稼働中」に変更 → OK

入庫・返却の手順

  1. アプリを開く
  2. 機材一覧 or 案件画面 → 対象機材をスキャン
  3. ステータスを「インターバル」に変更
  4. 整備が必要そう → 「故障中」 ／ 問題なし → 「出荷可能」

後半：清掃フロー・運用ディスカッション

進め方の方針

まず西口さんたちの現状フローをヒアリングすることから始める。「こちらが説明する」より先に「滋賀倉庫でどんな清掃・整備をしているか教えてもらう」姿勢で臨む。

ヒアリング（15分）

  • 日常的にどんな清掃・点検をしているか（項目・頻度・誰が担当）
  • 入庫したチラーにどんな作業をしているか（順番・時間感覚）
  • 困っていること・判断に迷う場面

清掃フロー設計の共有（20分）

分類定義承認
簡易清掃 日常清掃・チェック完了 不要
簡易メンテ プラグ・パッキン補充等、その場で対応できたもの 不要
修理要対応 その場ではどうにもならない不具合 古澤が最終判断

入庫時の清掃チェック項目（案）

🧊 チラー

  • 空冷フィン清掃（高圧洗浄機）
  • ストレーナー清掃
  • 内部清掃（ブライン使用時・極端に汚い場合）
  • マニュアル確認・補充（なければ新しいものを補充）
  • 外装清掃（極端に汚い場合のみ）
  • ラッシング緩みチェック
  • プラグ脱落確認・修正
  • カムロック（メス）パッキン脱落確認・補充
  • RFIDタグ確認 — なければ追加・紐付け。あるはずなのにない場合は要確認（→ 修理要対応）

迷ったときの3原則

  • 迷ったら「インターバル」（返却後）または「故障中」（要修理）に入れる。無理に正しいステータスを探さない
  • 分からなければ古澤に連絡（電話 / LINE）。どんな小さなことでもOK
  • 止まらず暫定で動く。ステータスは後から直せます
